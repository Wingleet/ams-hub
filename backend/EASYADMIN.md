# EasyAdmin - Administration Interface

## 🎯 Administration Access

The EasyAdmin administration interface is accessible via:

```
URL: http://localhost/admin
Login page: http://localhost/admin/login
```

## 🔐 Authentication

### Authorized Access

Only users with the following roles can access EasyAdmin:

- ✅ **ROLE_USER** - Standard users
- ✅ **ROLE_ADMIN** - Administrators

**Note** : Login is done via `/admin/login` (EasyAdmin), but standard users and administrators share the same login page. Access control is managed at the role level.

### Features

- **Secure login** - Authentication via EasyAdmin (Symfony Session)
- **Persistent session** - "Remember Me" for 1 week
- **Secure logout** - Logout button in menu
- **CSRF Protection** - CSRF token enabled for forms

## 👥 Create a User to Login

## 📄 Page Structure

### Main Dashboard

- **Route** : `/admin`
- **Controller** : `App\Controller\Admin\DashboardController`
- **Template** : Twig templates generated by EasyAdmin

### Login Page

- **Route** : `/admin/login`
- **Template** : `templates/security/login.html.twig` (EasyAdmin)

### Logout

- **Route** : `/admin/logout`
- Automatically redirects to `/admin/login`

## 📊 Entity Management

### Available CRUDs

The following entities have a complete EasyAdmin CRUD:

- **Users** (`User`) : User account management
    - Create/Edit/Delete users
    - Manage roles (ROLE_USER, ROLE_ADMIN)
    - Enable/Disable users

- **Organizations** : Organization management
    - Create/Edit/Delete organizations
    - Manage organization users
    - Manage subscriptions

- **Applications** : Application management
    - Create/Edit/Delete applications
    - Configure database_name
    - Enable/Disable applications

- **Subscriptions** : Subscription management
    - Create/Edit/Delete subscriptions
    - Set expiration dates
    - Manage active/inactive status

### Permissions

- ✅ All authenticated users (`ROLE_USER` or `ROLE_ADMIN`) can **access** EasyAdmin
- ✅ CRUDs are accessible with authentication
- ⚠️ POST/PUT/DELETE actions are controlled at API Platform level (generally require ROLE_ADMIN)

## 🔒 Security Configuration

### Firewall (`config/packages/security.yaml`)

```yaml
admin:
    pattern: ^/admin
    lazy: true
    provider: app_user_provider
    form_login:
        login_path: admin_login
        check_path: admin_login
        enable_csrf: true
    logout:
        path: admin_logout
        target: admin_login
    remember_me:
        secret: "%kernel.secret%"
        lifetime: 604800 # 1 semaine (en secondes)
```

### Access Control

```yaml
access_control:
    # Public - login not required
    - { path: ^/admin/login, roles: PUBLIC_ACCESS }

    # Admin - authentication required
    - { path: ^/admin, roles: [ROLE_USER, ROLE_ADMIN] }
```

## 🎨 Customization

### Change Dashboard Title

Dans `DashboardController::configureDashboard()` :

```php
return Dashboard::new()
    ->setTitle('AMS APPS HUB')
    ->setLocales(['fr', 'en']);
```

### Modify Menu

Dans `DashboardController::configureMenuItems()` :

```php
yield MenuItem::linkToDashboard('Dashboard', 'fa fa-home');
yield MenuItem::linkToCrud('User', 'fa fa-user', User::class);
yield MenuItem::linkToCrud('Organization', 'fa fa-building', Organization::class);
yield MenuItem::linkToCrud('Application', 'fa fa-th-large', Application::class);
yield MenuItem::linkToCrud('Subscription', 'fa fa-credit-card', Subscription::class);

yield MenuItem::section('Account');
yield MenuItem::linkToLogout('Logout', 'fa fa-sign-out');
```

## 🐛 Troubleshooting

### Erreur 401 Unauthorized

```bash
# Verify user exists
php bin/console doctrine:query:sql "SELECT * FROM \"user\" WHERE email='test@example.com'"

# Verify user is active (is_active = 1)
php bin/console doctrine:query:sql "SELECT id, email, is_active FROM \"user\""

# Verify password is correct (in database, it's hashed)
```

### Blank page or 500 error

```bash
# Clear cache
php bin/console cache:clear

# Check logs
tail -f var/log/dev.log

# Docker
docker compose exec backend php bin/console cache:clear
docker compose logs -f backend
```

### "Access Denied"

- Verify user has `ROLE_USER` or `ROLE_ADMIN` role
- Check `access_control` in `config/packages/security.yaml`
- Verify user is active (`is_active = 1`)

### Session doesn't persist

```bash
# Verify sessions are properly stored
php bin/console cache:clear --env=dev

# Check session configuration in framework.yaml
# session.cookie_secure and cookie_httponly must be enabled
```

## 🔗 Integration with Frontend API

The EasyAdmin interface is **independent** from the React frontend application. Both share:

- ✅ **Same database** (PostgreSQL)
- ✅ **Same authentication system** (EasyAdmin for admin, JWT API for frontend)
- ✅ **Same entities** (User, Organization, Application, Subscription)

The frontend (`/`) and EasyAdmin (`/admin`) are two distinct interfaces on the same project!
